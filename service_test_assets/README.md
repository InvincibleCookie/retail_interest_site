# Тестовые данные для автоматической проверки сервиса

Эта папка предназначена для ручного добавления файлов, которые будут использоваться скриптом `scripts/run_service_tests.py`.

## Что положить

```text
service_test_assets/
├── videos/      # положите сюда 1–2 настоящих видео: .mp4, .avi, .mov, .mkv или .m4v
├── archives/   # положите сюда ZIP-архив с одним или несколькими видео
└── invalid/     # необязательно: любые неподдерживаемые файлы для негативных проверок
```

Минимальный набор для запуска:

1. `service_test_assets/videos/single_valid_video.mp4` — короткое видео, которое открывается OpenCV.
2. `service_test_assets/archives/videos.zip` — ZIP-архив, внутри которого есть хотя бы одно видео поддерживаемого формата.

## Как запустить

Из корня проекта:

```bash
python scripts/run_service_tests.py
```

По умолчанию скрипт проверяет именно сервисный слой и веб-API: загрузку видео, загрузку ZIP, получение первого кадра, запуск обработки, получение статуса, контракты кнопок интерфейса, получение отчета и скачивание ZIP-отчета. После запуска Markdown/JSON-отчет сохраняется в `data/test_reports/`. ML-пайплайн при этом заменяется детерминированной тестовой заглушкой, чтобы проверка не требовала GPU, `.npz`-признаков и весов модели.

Если нужно прогнать проверку с настоящим ML-пайплайном, заранее подготовьте признаки и модель из `README.md`, затем запустите:

```bash
python scripts/run_service_tests.py --real-ml
```

В режиме `--real-ml` подробный вывод пайплайна скрывается, чтобы итог был компактным. Для полного лога запустите:

```bash
python scripts/run_service_tests.py --real-ml --verbose-ml
```
