# Retail Interest Zones

Веб-приложение для автоматического анализа поведения покупателей у полки. Пользователь загружает видео или ZIP-архив, выделяет область полки и запускает обработку. Сервис самостоятельно извлекает признаки из исходного видео, классифицирует действия и формирует тепловую карту зон интереса.

## Возможности

- загрузка `.mp4`, `.avi`, `.mov`, `.mkv`, `.m4v` и ZIP-архивов;
- обработка каждого загруженного видео непосредственно с исходных кадров;
- извлечение позы человека моделью `yolov8m-pose.pt`;
- извлечение RGB-признаков моделью EfficientNet-B0;
- выбор активных кадров по результатам классификации и извлечение 133 WholeBody-точек моделью RTMW;
- классификация действий моделью `best_C6_LateFusion_Gated_BiGRU.pt`;
- ручное выделение ROI полки рамкой или полигоном;
- построение тепловой карты событий рук и ранжирование зон интереса;
- формирование HTML, Excel, CSV и ZIP-отчетов.

## Полный пайплайн

Для каждого видео автоматически выполняются следующие этапы.

### 1. YOLOv8m-pose

Модель `yolov8m-pose.pt` обрабатывает все кадры и возвращает bounding box человека, confidence и 17 ключевых точек COCO. Если найдено несколько людей, выбирается кандидат с максимальным значением площади рамки, умноженной на confidence.

Координаты ключевых точек нормализуются относительно центра и размера bounding box. Пропуски интерполируются, координаты сглаживаются медианным окном, после чего добавляются 11 расстояний между плечами, локтями, запястьями и бедрами. Размер frame-level вектора равен 68.

Кадры группируются по 6. Для каждого чанка рассчитываются `mean`, `std`, `dx_mean` и `dx_std`. Итоговая размерность pose-признаков равна 272.

### 2. RGB-признаки

Каждый кадр преобразуется в RGB и проходит через предобученную EfficientNet-B0 без классификационного слоя. На кадр получается вектор размерности 1280. После агрегации по 6 кадров через `mean`, `std`, `dx_mean` и `dx_std` размерность RGB-признаков равна 5120.

### 3. Классификация действий

Pose-признаки размерности 272 и RGB-признаки размерности 5120 нормализуются перед передачей в `best_C6_LateFusion_Gated_BiGRU.pt`. Параметры `mean` и `std` сервис рассчитывает по chunk-признакам каждого загруженного видео.

Normalizer сохраняется по полному пути внутри репозитория:

```text
<корень_репозитория>/data/outputs/<job_id>/features/normalizers/<video_id>_fusion_normalizers.npz
```

Модель классифицирует каждый шестикадровый чанк по классам:

```text
background
reach_to_shelf
retract_from_shelf
hand_in_shelf
inspect_product
inspect_shelf
```

### 4. Выбор кадров и RTMW WholeBody

После классификации сервис выбирает только чанки, у которых сигнал действий, связанных с взаимодействием с полкой, достигает порога `raw_min_action_signal`. В список RTMW включаются кадры только из этих чанков. Поэтому YOLO person gate и RTMW WholeBody не запускаются на фоновых участках видео.

На выбранных кадрах YOLO находит человека, bounding box расширяется и передается в RTMW WholeBody. Для последовательных выбранных кадров рамка и ключевые точки сглаживаются; при разрыве последовательности состояние сглаживания сбрасывается. RTMW возвращает 133 точки:

- `0..16` — тело COCO;
- `17..22` — стопы;
- `23..90` — лицо;
- `91..111` — левая кисть;
- `112..132` — правая кисть.

В файл сохраняются только результаты выбранных кадров: `keypoints`, `scores`, `boxes` и исходные номера кадров `frame_indices`. Это уменьшает число запусков RTMW и размер промежуточного файла, сохраняя данные, которые используются для локализации кистей.

### 5. Зоны интереса

RTMW-точки кистей объединяются с вероятностями действий. События фильтруются по выносу руки, confidence и силе действия, затем проецируются в ROI полки. Взвешенные точки формируют heatmap. После адаптивной пороговой обработки и поиска связных компонент формируются зоны интереса, для которых рассчитываются:

- суммарный и пиковый интерес;
- количество событий;
- количество активных кадров;
- главное действие;
- нормализованная оценка интереса от 0 до 100.

## Требования

- Python 3.10 или новее;
- CUDA-совместимая видеокарта рекомендуется для ускорения YOLO, EfficientNet и RTMW;
- файл обученной модели `best_C6_LateFusion_Gated_BiGRU.pt`.

Модели YOLO, EfficientNet-B0 и RTMW загружаются библиотеками автоматически при первом запуске.

## Установка

```bash
python -m venv .venv
```

Windows:

```powershell
.venv\Scripts\activate
pip install -r requirements.txt
```

Linux/macOS:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

Для запуска RTMW на NVIDIA GPU вместо CPU установите GPU-вариант ONNX Runtime:

```bash
pip uninstall -y onnxruntime
pip install onnxruntime-gpu
```


https://drive.google.com/drive/folders/1d1Iq61Xhxaunt18o7tiM6pYXbJM3ZXj6?usp=sharing

Linux/macOS:

```bash

```

## Прогресс обработки

Во время анализа интерфейс показывает:

- текущую стадию: YOLOv8m-pose, EfficientNet-B0, RTMW WholeBody, нормализация, классификация, построение зон или формирование отчета;
- процент выполнения, рассчитанный по обработанным кадрам и количеству видео;
- устройство выполнения: CPU либо название CUDA-видеокарты;
- прошедшее время и расчетное время до завершения.

Оценка оставшегося времени уточняется по мере обработки кадров, поэтому в начале запуска она может изменяться заметнее.

## Запуск

Windows:

```powershell
run_server.bat
```

Linux/macOS:

```bash
./run_server.sh
```

Откройте в браузере:

```text
http://localhost:8000
```

## Использование

1. Загрузите одно или несколько видео либо ZIP-архив.
2. Выберите видео в списке.
3. Нарисуйте ROI полки рамкой или полигоном.
4. Нажмите `Применить ко всем`, если положение камеры одинаково для всех видео.
5. Нажмите `Запустить анализ`.
6. Дождитесь завершения извлечения признаков и классификации действий.
7. Откройте HTML-отчет или скачайте итоговый ZIP.

## Настройка моделей извлечения

Значения по умолчанию:

```text
yolo_pose_model = yolov8m-pose.pt
yolo_person_model = yolo11n.pt
yolo_imgsz = 640
yolo_pose_conf = 0.15
yolo_person_conf = 0.30
rgb_batch_size = 32
chunk_size = 6
```

Повторная обработка одного задания может использовать уже извлеченные pose- и RGB-признаки. RTMW-файл формируется заново после классификации, поскольку набор кадров зависит от рассчитанного сигнала действий и параметра `raw_min_action_signal`. Для принудительного повторного извлечения pose/RGB можно передать параметр `force_feature_extraction=true` в API обработки.
